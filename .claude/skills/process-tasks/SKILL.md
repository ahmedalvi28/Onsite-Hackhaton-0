# Process Tasks Skill

## Description
Processes all pending task files in the Needs_Action folder and creates corresponding plan files in the Plans folder.

## Usage
Use this skill when:
- New tasks have been dropped into the Drop folder and moved to Needs_Action
- You want to batch process multiple pending tasks
- The watcher has detected new items and created task files

## Parameters
- `vault_path`: Path to Obsidian vault (default: C:/Users/alvia/OneDrive/Desktop/AI_Employee_Vault_Hackathon)

## Expected Actions

1. **Read all pending tasks** from Needs_Action folder
2. **Categorize each task** as business or personal
3. **For each task**:
   - Read the task file content
   - Determine task type and requirements
   - Generate a comprehensive Plan.md file
   - Place Plan.md in Plans folder
   - Move task to In_Progress folder
   - Log the action

4. **Update dashboard** with task counts

## Plan.md Template

```markdown
---
task_id: TASK_ID
task_type: task_type
status: pending_approval
created: timestamp
---

# Plan: [Task Title]

## Task Overview
[Brief description of what needs to be done]

## Requirements
- Requirement 1
- Requirement 2
- ...

## Steps to Complete

1. [ ] Step 1
2. [ ] Step 2
3. [ ] ...

## Resources Needed
- Resource 1
- Resource 2

## Notes
[Any additional notes or considerations]

---
*Generated by Process Tasks Skill*
```

## Example

User: /process-tasks

Response: "Processed 3 pending tasks. Created 3 plan files in Plans folder."
